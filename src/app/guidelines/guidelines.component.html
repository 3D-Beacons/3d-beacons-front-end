Guidelines for Collaborating

Introduction

3D-Beacons is an open collaboration between providers of macromolecular structure models. This collaboration aims to provide model coordinates and meta-information from all the contributing data resources in a standardised data format and unified platform.

Terms of Collaboration

PDBe-KB

  1. PDBe-KB will maintain data exchange format schema(s)
  2. The schema will evolve in consultation with partner data resources
  3. PDBe-KB will support the infrastructure for the 3D-Beacons Hub and Registry
  4. The 3D-Beacons Hub will provide programmatic access to models of partner data resources
  5. The 3D-Beacons Hub will link back to the partner data resources, transparently attributing credit for their contributions
  6. PDBe-KB will maintain an open-access library of reusable data visualisation components
  7. PDBe-KB and the collaborating partners define a transparent ranking of the models based on agreed criteria

Collaborating partners

  1. The data contributed to 3D-Beacons by collaborating partners will be free from any restrictions on distribution and re-use under the CC BY 4.0 license or CC BY SA 4.0.
  2. The partners are responsible for the quality of the data they contribute
  3. Protocols for data generation of theoretical models must be published in peer-reviewed publications
  4. The partners commit to updating the API implementations within two months of announcing major version updates of the API specification.
    1. The API specification uses semantic versioning, e.g. v.1.3.9, where 1. is the major version, 3. is the minor version and 9. is a patch.
    2. The Hub API will aggregate the current (live) API endpoints.
    3. The development Hub API will test the development API endpoints.
  5. The partners commit to keeping Beacon API response times under 2 seconds for pre-calculated data.
    1. The 3D-Beacons Hub API enforces a 5 seconds timeout, i.e. Beacons not returning data within 5 seconds will not have their data exposed.
    2. New partner resources will have their API stress-tested by PDBe-KB.
