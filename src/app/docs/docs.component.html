The 3D-Beacons Network provides unified programmatic access to experimentally determined and predicted structure models.

Accessing data programmatically

The purpose of the 3D-Beacons Network is to facilitate programmatic access to predicted and experimental protein structures. To support this, we provide an extensive Google Colab tutorial illustrating how to interact with the network programmatically. This interactive guide offers practical examples, details different API endpoints, and showcases many potential use cases for your exploration.

Infrastructure

The 3D-Beacons Network consists of a Registry, a Hub and data providers who host Clients. The 3D-Beacons Hub uses the 3D-Beacons Registry to look up which 3D-Beacon Client supports what API endpoints. All Beacons APIs as well as the Beacons Hub provide data according to the 3D-Beacons data specifications. The Hub collates the data from the Beacons and exposes it via Hub API endpoints.

Currently, 3D-Beacons Hub provides URLs to, and description of every protein structure for a UniProt accession. For example: https://www.ebi.ac.uk/pdbe/pdbe-kb/3dbeacons/api/uniprot/summary/P0DTD1.json

The 3D-Beacons Hub API endpoints, such as shown above, can be used to programmatically access all the available protein structures and models from any members of the 3D-Beacons Network. For example, the API endpoint for the SARS-CoV-2 polyprotein, identified by its UniProt accession P0DTD1 provides all the experimentally determined structures from the PDB, as well as theoretical models from SWISS-MODEL and others.

Components of the 3D-Beacons infrastructure

3D-Beacons API specification

The main purpose of 3D-Beacons is to provide programmatic access to experimentally determined and theoretical protein structures in a uniform manner. We have created a set of data exchange format specifications to ensure the uniformity of the data provided by all member Beacons.

The specification is described on GitHub at https://github.com/3D-Beacons/3d-beacons-specifications (main branch)

3D-Beacons Registry

The 3D-Beacons Registry contains information on which data providers are members of the 3D-Beacons Network, and which API specifications did they implement. This allows both the 3D-Beacons Hub and any users to find out what API endpoints are available from which data providers. For example, if users are only interested in getting data from SASBDB, they can check in the registry which API URLs they should use.

The 3D-Beacons Registry is available at https://github.com/3D-Beacons/3d-beacons-registry

3D-Beacons Client

The 3D-Beacons Client is a lightweight Python package that allows researchers to link their in-house models to the 3D-Beacons Network. It takes PDB and/or mmCIF files as input and generates the necessary metadata which is then exposed through a set of API endpoints that comply with the 3D-Beacons data specifications.

The 3D-Beacons Client is publicly available at https://github.com/3D-Beacons/3d-beacons-client

3D-Beacons Hub

The 3D-Beacons Hub acts as an aggregator of data from the various members of the 3D-Beacons Network. It uses the 3D-Beacons Registry to look up which API endpoints are supported by which 3D-Beacons, retrieves the data, and collates it to present the combined metadata programmatically to users.

The 3D-Beacons Hub source code is available at https://github.com/3D-Beacons/3d-beacons-hub-api
