Background

3D-Beacons is an open collaboration between providers of macromolecular structure models. This collaboration aims to provide model coordinates and meta-information from all the contributing data resources in a standardised data format and unified platform.

Data providers

Providers of experimentally determined structures and predicted structures both distribute data through 3D-Beacons. Currently, the following data providers make their protein structures and annotations accessible:

Experimentally determined structures

  • Protein Data Bank in Europe
  • Small Angle Scattering Biological Data Bank

Deep learning models

  • AlphaFold Protein Structure Database

Template-based models

  • SWISS-MODEL
  • Genome3D

Conformational ensembles

  • Protein Ensemble Database

Annotations

  • Protein Data Bank in Europe - Knowledge Base

Infrastructure

The 3D-Beacons Network consists of a Registry, a Hub and data providers who host Clients. The 3D-Beacons Hub uses the 3D-Beacons Registry to look up which 3D-Beacon Client supports what API endpoints. The Beacons provide data according to the 3D-Beacons data specifications. The Hub collates the data from the Beacons and exposes it via Hub API endpoints.

Currently, 3D-Beacons provide the following information:

  • URLs to, and description of every protein structure for a UniProt accession, for example,
    https://3d-beacons.org/api/uniprot/summary/P0DTD1.json
  • Functional annotations from PDBe-KB for a UniProt accession, for example,
    https://3d-beacons.org/api/uniprot/annotations/P0DTD1.json
