The 3D-Beacons Network provides unified programmatic access to experimentally determined and predicted structure models.

Background

3D-Beacons is an open collaboration between providers of macromolecular structure models. This collaboration aims to provide model coordinates and meta-information from all the contributing data resources in a standardised data format, on a unified platform.

All of the data provided is freely available for both academic and commercial use under Creative Commons Attribution 4.0 (CC-BY 4.0) license terms. If you use this resource, please cite the following papers: M. Varadi, S. Nair, I. Sillitoe et al. 3D-Beacons: decreasing the gap between protein sequences and structures through a federated network of protein structure data resources. GigaScience (2022).

The 3D Beacons API retrieves and combines data from member data providers, and returns the data in JSON format according to the 3D-Beacons API specification.

How to use it?

The main purpose of 3D-Beacons is to provide programmatic access to experimentally determined and theoretical protein structures.

The 3D Beacons API retrieves and combines data from member data providers, and returns the data in JSON format according to the 3D-Beacons API specification.

How does it work?

The 3D-Beacons Network consists of a Registry, a Hub and data providers who host Clients. The 3D-Beacons Hub uses the 3D-Beacons Registry to look up which 3D-Beacon Client supports what API endpoints. The Beacons provide data according to the 3D-Beacons data specifications. The Hub collates the data from the Beacons and exposes it via Hub API endpoints.

Case studies of integrating 3D-Beacons

Jalview

3D Beacons allows users of the Jalview workbench for multiple sequence alignment and analysis to discover 3D models for alignments of proteins from Uniprot and place them in context of genetic variation from Ensembl. Its capabilities also include visualisation of local model quality scores (including method specific quantities such as pLDDT) and sequence alignment based superimposition of structures with Jmol, UCSF Chimera/X and Pymol.

For more information see Jalview's 3D Beacons search documentation. The Jalview application is available for a variety of platforms and can also be run in the browser.

PDBe-KB

The Protein Data Bank in Europe - Knowledge Base (PDBe-KB) relies on the 3D-Beacons Hub API to display all the experimentally determined and theoretical structures for proteins of interest (example). The so-called "aggregated views of proteins" pages of PDBe-KB collate all the available structural data from the PDB and place these structures in the context of their functional, biophysical and biochemical annotations provided by partner data resources. The integration of the 3D-Beacons Hub API allows us to display these annotations concerning theoretical models and the experimentally determined structures.

SWISS-MODEL Repository

The SWISS-MODEL Repository (SMR) takes advantage of the 3D-Beacons API to fetch models from the AlphaFoldDB and ModelArchive for display in addition to homology models from SWISS-MODEL and experimental structures from the PDB (example). Within SMR, the models are displayed with a consistent colouring for model confidence to compare homology with AlphaFold models and structures can be further assessed with other metrics. Furthermore, SMR includes sequence based annotations which can be visualised on the structures and enables the comparison of multiple structures.
